# RFPilot v3 — Full Project Documentation

AI-powered RFP (Request for Proposal) management platform. Backend: FastAPI + SQLAlchemy + PostgreSQL/pgvector. Frontend: React 18 + Vite + React Router.

Generated by reading every source file in the repository (excluding `node_modules/`, `venv/`, `__pycache__/`, `.git/`). Binary/build artifacts and lockfiles are noted but not exhaustively detailed.

## 1. BACKEND — Core App (`starter/`)

📄 **starter/main.py**
   Purpose    : FastAPI application entry point; wires up CORS and registers every router.
   Contains   : `app = FastAPI()`; `GET /` health/status route; CORS middleware (allows `localhost:3001`, `localhost:5173`); includes `register_router`, `login_router`, `rfp_router`, `notif_router`, `deadline_router`, `invitation_router`, `team_router`, `dashboard_router`.
   Connects to: imports `database` (engine, Base) and `models`; imports routers from `register.py`, `login.py`, `rfp_routes.py`, `notification_routes.py`, `invitation_routes.py`, `team_routes.py`, `dashboard_routes.py`. This is the file `uvicorn` points at.
   Used when  : Application startup (`uvicorn main:app`).
   Critical   : Yes — without it nothing is wired together.

📄 **starter/database.py**
   Purpose    : Database connectivity setup — engine, session factory, declarative base, FastAPI DB dependency.
   Contains   : `DATABASE_URL` (from env, with hardcoded local fallback), `engine` (`pool_pre_ping=True`), `SessionLocal`, `Base`, `get_db()` generator dependency.
   Connects to: used by every route/model file via `from database import get_db`/`Base`; loaded by `alembic/env.py`.
   Used when  : App startup (engine init) and on every request (`Depends(get_db)`).
   Critical   : Yes.

📄 **starter/models.py**
   Purpose    : All 18 SQLAlchemy ORM models defining the full schema (auth, RFP lifecycle, AI analysis, RAG, collaboration, drafts).
   Contains   : `User`, `RFP`, `RFPInformation`, `RFPDocument`, `AISummary`, `TechnicalAnalysis`, `BidDecision`, `Risk`, `EvaluationCriteria`, `DocumentChunk` (pgvector 1536-dim embedding), `AIChatHistory`, `Resource`, `ApprovalStep`, `Comment` (self-referential replies), `Notification`, `Invitation`, `DraftDocument`, `DraftVersion`, `Requirement`, `RFPDeadline`.
   Connects to: imports `database.Base`, `pgvector.sqlalchemy`; imported by every route file and `alembic/env.py`.
   Used when  : Schema definition at import time; ORM queries at request time.
   Critical   : Yes.

📄 **starter/schemas.py**
   Purpose    : Pydantic request/response contracts for the API.
   Contains   : `RegisterRequest`, `LoginRequest`, `UserResponse`, `RegisterResponse`, `TokenResponse`, `GenerateDraftRequest`, `ChatRequest`, `ChatResponse`, `IngestRequest`, `CommentCreate`, `DecisionUpdate`, `InvitationCreate`, `InvitationAccept`, `RoleUpdate`.
   Connects to: imported by `login.py`, `register.py`, `rfp_routes.py`, `invitation_routes.py`, `team_routes.py`.
   Used when  : Every request validation / response serialization.
   Critical   : Yes.

📄 **starter/auth_utils.py**
   Purpose    : JWT + password-hashing utilities and the `get_current_user` auth dependency.
   Contains   : `SECRET_KEY`, `ALGORITHM="HS256"`, `ACCESS_TOKEN_EXPIRE_MINUTES=1440`, `pwd_context` (bcrypt), `oauth2_scheme`, `hash_password()`, `verify_password()`, `create_access_token()`, `decode_access_token()`, `get_current_user()`.

📄 **starter/login.py**

📄 **starter/register.py**
   Purpose    : User registration endpoint.

   Connects to: `auth_utils.hash_password`, `database.get_db`, `models.User`, `schemas`. Registered in `main.py`.

📄 **starter/dashboard_routes.py**

## 11. End-to-End Flow Summary (how the layers connect)

1. **Browser loads** `index.html` → `src/main.jsx` → `src/App.jsx`, which mounts `AuthProvider` and the router.
2. **Unauthenticated user** hits `/login` or `/register` → `src/api/auth.js` calls backend `POST /auth/login` or `/auth/register` (`starter/login.py` / `register.py`) → backend hashes/verifies via `auth_utils.py`, queries `models.User` through `database.py` → returns a JWT → frontend stores it in `localStorage` and sets it on the Axios instance (`src/api/api.js`/`src/services/api.js` interceptor).
3. **`ProtectedRoute`** gates `/dashboard`, `/upload-rfp`, `/detail`, `/team` behind a valid `AuthProvider` user; `Layout.jsx` wraps them with `ChatProvider` + the global `AIChat` panel and starts `useNotificationStream`.
4. **Dashboard** (`Dashboard.jsx`) calls `dashboardApi.getAnalytics()` → `starter/dashboard_routes.py` aggregates `RFP`, `User`, `Notification`, `RFPDeadline`, `AISummary`, `BidDecision` rows → rendered via `KPISection`, `TrackingTabs`, `BidDecisionCards`, `RightPanel`, filtered through `FilterBar`/`data/constants.js`.
5. **Upload** (`UploadRFPPage.jsx`) posts a file to `rfpApi.upload()` → `starter/rfp_routes.py`'s `POST /rfps/upload` extracts text (PyMuPDF), calls OpenAI for structured extraction, persists `RFP`/`RFPInformation`/`RFPDocument`/`AISummary`/`Requirement`/`EvaluationCriteria`/`RFPDeadline`/`ApprovalStep`/`BidDecision`, and chunks+embeds the document via `rag_engine.ingest_document()` into `DocumentChunk` (pgvector).
6. **Detail page** (`DetailPage.jsx`) loads `GET /rfps/{id}` and renders `OverviewTab`/`TechnicalTab`/`DecisionTab`/`CollaborationTab`, each calling its own slice of `rfp_routes.py` endpoints (requirements, criteria, workflow, drafts, comments, decision).
7. **AI Chat** (`AIChat.jsx`) posts to `rfpApi.chat()` → `POST /rfps/chat` → `rag_engine.chat()` runs a tool-calling loop against gpt-4o, retrieving relevant `DocumentChunk`s via pgvector similarity search, optionally calling `get_rfp_count`/`list_rfps`/`search_rfp_content`, and logs the exchange to `AIChatHistory`.
8. **Notifications/Deadlines** flow from backend events (status changes, comments, workflow updates) into the `Notification`/`RFPDeadline` tables, surfaced via `notification_routes.py`'s SSE stream (`useNotificationStream.js`) and polled REST endpoints (`RightPanel`, `RightSidebar`).
9. **Team/Invitations**: an admin uses `TeamPage.jsx`/`InviteModal.jsx` → `invitationApi`/`teamApi` → `starter/invitation_routes.py`/`team_routes.py`, which use `email_utils.py` to send invite emails (or return a manual link if SMTP isn't configured); the invitee lands on `AcceptInvitePage.jsx` → `POST /invitations/{token}/accept` creates/links their `User` row.
10. **Schema** for all of the above is defined in `starter/models.py` and built up incrementally by the 11 Alembic migrations in `starter/alembic/versions/`, run via `alembic upgrade head` against the `DATABASE_URL` in `starter/.env`.

### Notable issues flagged during this audit
- **Dead/legacy code**: `starter/store.py`, `starter/store_rfp.py`, `starter/bot.py`, `starter/routers/auth.py`, `starter/routers/evaluate.py` (the last two are unfinished TODO stubs) are not wired into `main.py` and can likely be deleted.
- **Misplaced file**: `starter/package.json`/`package-lock.json` is a stray Node manifest inside the Python backend with a single unused `vite` dependency.
- **No `.gitignore` at repo root** — `node_modules/`, `venv/`, `__pycache__/`, `*.pyc`, `dist/`, and SQLite `.db`/log files are all tracked in git, which is almost certainly unintentional.
- **Stale data artifacts**: `starter/auth.db`, `starter/rfps.db`, `starter/lesson9.db`, `starter/chroma_data/chroma.sqlite3`, `frontend.log`, `starter/backend.log`.
- **Outdated manual test scripts** at repo root (`test_auth.py`, etc.) reference an old request schema (`username`/`buyer` role) that no longer matches current `RegisterRequest`.
- **`dist/` contains two sets of hashed build bundles**, suggesting an uncleaned previous build was committed alongside the latest one.
